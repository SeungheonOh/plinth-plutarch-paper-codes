# Synthetic error-message rig

Each `EXX-*/` directory holds a **minimal, self-contained** Plinth/Plutarch
source pair that triggers one error from `../error-message-comparison.md`
when compiled. There is no patch step here — the `.hs` files **are** the
test cases. Each file deliberately introduces a single mistake (division
on Integer, missing constructor, captured TH variable, etc.) so that the
captured GHC diagnostic mirrors what a beginner would see writing similar
code from scratch.

The companion `../src-errormsg-top8/` directory contains the eight
top-distinct error cases recreated **inside the real validators under
`src/`** via unified-diff `.patch` files. Pick whichever lens the paper
needs:

| Goal | Use |
|------|-----|
| Smallest reproducer of a single mistake | `src-errormsg/EXX-*/` |
| Same mistake injected into production-shaped code | `src-errormsg-top8/EXX-*/` |

## Layout

```
src-errormsg/
  E01-…/  …  E35-…/
    Plinth.hs   ← minimal toy producing the bug (the test case itself)
    Plutarch.hs ← matching Plutarch toy (where the bug has a Plutarch analogue)
    Plinth.err  ← captured GHC diagnostic (regenerated by collect-errors.sh)
    Plutarch.err
  collect-errors.sh
  .gitignore
```

## Workflow

```bash
nix develop
src-errormsg/collect-errors.sh           # compile every case, capture diagnostics
src-errormsg/collect-errors.sh E04 E18   # restrict to specific cases
```

`collect-errors.sh` invokes `cabal exec ghc -- -Wall -fforce-recomp -c …` on
every `EXX/Plinth.hs` and `EXX/Plutarch.hs`. The filtered output (cabal/nix
preamble stripped) lands in the sibling `.err` file.

The script disambiguates `Plinth.Plugin` by reading the unit-id cabal
pinned for `plutus-tx-plugin` from the current `GHC_ENVIRONMENT` and
passing it as `-package-id`. Without that step GHC sees multiple
`plutus-tx-plugin-1.64.0.0` candidates in the cabal store (from prior
`cabal build` runs) and refuses to choose.

Per-case verdict is one of:

* `errors captured`        — GHC failed (expected for most cases).
* `warnings captured`      — GHC succeeded but emitted at least one warning
                             (expected for E17, E18, E23 under `-Wall`).
* `clean (no diagnostics)` — neither errors nor warnings — the bug
                             didn't fire.

## Captured diagnostics

Every captured diagnostic is written to `EXX-*/<Variant>.err`, sitting
next to the source file that produced it. Nothing is written outside the
case directory.

For the eight cases curated in the paper's top-8 set:

```
src-errormsg/E04-DivisionOnIntegers/{Plinth,Plutarch}.err
src-errormsg/E05-BoolForInteger/{Plinth,Plutarch}.err
src-errormsg/E12-MissingAppParens/{Plinth,Plutarch}.err
src-errormsg/E18-MissingConstructor/{Plinth,Plutarch}.err
src-errormsg/E21-WrongPreludeFunction/{Plinth,Plutarch}.err
src-errormsg/E25-InfiniteType/{Plinth,Plutarch}.err
src-errormsg/E27-UnsupportedGADTs/Plinth.err
src-errormsg/E29-StageError/Plinth.err
```

…each documents the same diagnostic signature as the matching real-
validator case under `src-errormsg-top8/`, just on a much smaller program.
